01

The Trinity Pattern

Every AdvanxJS component is exactly three files — no exceptions. Logic lives in TypeScript, markup in HTML, styles in CSS. They never mix.

logic.ts

import { signal } from "advanxjs";

export const count = signal(0);

export function increment() {
  count.value++;
}

view.html

<p>Count: {{ count }}</p>

<button ax-on:click="increment">
  Add
</button>

style.css

p {
  font-size: 1.5rem;
  font-weight: 700;
  color: var(--ax-brand);
}

button { cursor: pointer; }

The compiler validates every binding at build time. If view.html references a signal or function that isn't exported from logic.ts, the build fails loudly. No runtime surprises.

02

Signals

AdvanxJS uses three reactive primitives from @preact/signals-core. All state flows through signals — never through imperative DOM manipulation.

signal(initialValue)

Writable reactive value. Read and write via .value. Any change triggers DOM updates automatically.

computed(() => expr)

Read-only derived value. Recomputes automatically when its dependencies change. Cannot be written to.

effect(() => fn)

Side-effect runner. Re-executes whenever its signal dependencies change. Used internally by the runtime to wire DOM updates.

typescript — logic.ts
import { signal, computed } from "advanxjs";

// Writable signal — user toggles this
export const isYearly = signal(false);

// Computed — derives from isYearly automatically
export const proPrice = computed(() =>
  isYearly.value ? "$290/yr" : "$29/mo"
);
export const period = computed(() =>
  isYearly.value ? "/year" : "/month"
);

// Action — mutates the signal
export function toggleBilling() {
  isYearly.value = !isYearly.value;
}

Rule: computed values are read-only in the view. You cannot bind ax-model to a computed — only to a writable signal.

03

Directives Reference

Directives are HTML attributes that bind the view to logic. All values must be bare identifiers — no expressions, no ternaries, no inline JS.

Directive Syntax Purpose Constraint
{{ }} {{ count }} Text interpolation — replaces the expression with the signal's current value Signal or computed only. Nested paths (e.g. item.label) supported inside ax-for loops.
ax-if ax-if="isOpen" Conditionally mounts/unmounts the element. Falsy → replaced by a comment placeholder. Must be a boolean signal or computed. No expressions.
ax-for ax-for="item in items" Renders the element once per array item. Alias (item) is available in mustaches inside the element. Source must be a signal holding an array.
ax-on:EVENT ax-on:click="submit" Attaches a DOM event listener. Calls the named exported function when the event fires. Value must be an exported function name, not a signal.
ax-model ax-model="email" Two-way binding for <input> and <textarea>. Syncs on input and change events. Writable signals only — not computed.
ax-link ax-link="/about" SPA navigation. Updates the URL without a full page reload. Triggers popstate for back/forward. Static path string. Dynamic paths must use window.location.href in logic.
html — view.html — all directives in one component
<!-- Mustache -->
<p>Hello, {{ username }}!</p>

<!-- ax-if -->
<div ax-if="isLoggedIn">Dashboard</div>

<!-- ax-for -->
<li ax-for="item in items">{{ item.label }}</li>

<!-- ax-on:click -->
<button ax-on:click="logout">Sign out</button>

<!-- ax-model -->
<input ax-model="email" type="email" />

<!-- ax-link -->
<a ax-link="/profile">My Profile</a>
04

CLI Reference

The advanx CLI handles scaffolding, building, serving, and adding registry components.

Command Usage Description
create advanx create my-app Scaffold a new AdvanxJS project with the counter starter template.
build advanx build src/components/navbar Compile a single component. Validates bindings, generates .advanx-meta.json, bundles to dist/bundle.js.
dev advanx dev src/components/navbar Watch mode — rebuilds on file changes with live reload.
serve advanx serve dist Static file server with SPA history fallback. No Vite, no Webpack — just Bun.
add advanx add navbar Download a component from the registry into src/components/<name>/ and compile it immediately.
add --list advanx add --list Print all available registry components with their tier (free / pro).
explain advanx explain src/components/counter Print the component's signals, computed values, actions, and bindings — structured for pasting into an AI assistant.

.advanx-meta.json

Every compiled component automatically emits a .advanx-meta.json file — a structured map of its signals, computed values, actions, and bindings. The build fails if this file cannot be generated. Run advanx explain src/components/counter to print it to your terminal and paste it directly into an AI assistant.

json — .advanx-meta.json (counter example)
{
  "component": "counter",
  "signals": ["count"],
  "computed": [],
  "actions": ["increment"],
  "structure": {
    "mustaches": ["count"],
    "conditionals": [],
    "events": [{ "event": "click", "handler": "increment" }],
    "loops": [],
    "models": []
  },
  "tokens_hint": "This component is AdvanxJS compliant. Logic and View are decoupled."
}
